# Metodología de análisis de vulnerabilidades y desarrollo de exploits

Este repositorio documenta la creación de un entorno de laboratorio para el análisis de vulnerabilidades y el desarrollo de exploits.

El objetivo del proyecto es mostrar una metodología estructurada para investigar vulnerabilidades en software real, preparar un entorno de análisis adecuado y desarrollar pruebas de concepto que permitan comprender el proceso de explotación.

El laboratorio incluye herramientas de reversing, debugging y explotación utilizadas habitualmente en investigación de vulnerabilidades.

## Estructura del proyecto

El repositorio está dividido en dos partes principales.

### Entorno de investigación

En esta sección se documenta la preparación del entorno de laboratorio utilizado para el análisis de vulnerabilidades. Se describen las herramientas instaladas y su función dentro del proceso de investigación.

Las herramientas incluyen:

- Lenguajes de programación
- Entornos de desarrollo
- Debuggers y disassemblers
- Herramientas de explotación
- Aplicaciones vulnerables
- Herramientas de red

La documentación completa puede encontrarse en la carpeta `entorno`.

### Análisis y explotación de vulnerabilidades

En esta sección se describe la metodología utilizada para analizar vulnerabilidades y desarrollar exploits.

También se incluye un caso práctico basado en la vulnerabilidad **CVE-2025-5548**, donde se estudia el comportamiento vulnerable de la aplicación y el proceso de explotación.

La documentación completa se encuentra en la carpeta `explotacion`.

## Objetivo del proyecto

El objetivo de este proyecto es demostrar una metodología técnica clara para el análisis de vulnerabilidades, combinando investigación, análisis técnico y documentación estructurada.
